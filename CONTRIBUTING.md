## How to contribute

### Reporting bugs

If you find a bug in FEelMRI, please report it on the [GitHub issue
tracker](https://github.com/hmella/feelmri/issues/new?labels=bug).


### Suggesting enhancements

If you want to suggest a new feature or an improvement of a current
feature, you can submit this on the [issue
tracker](https://github.com/hmella/feelmri/issues).


### Submitting a pull request

To contribute code FEelMRI, create a pull request. Before submitting a pull
request, please make sure that your code follows the existing coding style
and that you have added unit tests for any new functionality.

On opening a pull request, unit tests will run on GitHub CI. You can
click on these in the pull request to see where (if anywhere) the tests
are failing.
